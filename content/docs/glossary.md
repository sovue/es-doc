# Глоссарий

Во время работы над модификацией мододелы часто приходят к использованию различных сокращений и жаргонных слов, которые используют только они. Обычному читателю или новичку некоторые из них могут быть непонятны, поэтому на этой странице будут описаны самые известные из них.

## Основные термины

### Дисплеебл

**Дисплеебл** (от англ. *displayable* — отображаемость) — любой визуальный объект, который можно показать пользователю.

### Экран

**Экран** (он же *screen*) — способ отображения пользовательского интерфейса (среды с помощью которой пользователь взаимодействует с программой) в Ren'Py.

### Стиль

**Стиль** — объект, хранящий свойства элементов интерфейса, таких как текст, кнопки, рамки и т.п.

### Переход

**Переход** (он же *transition*) — способ применения различных эффектов при отображении и скрытии дисплееблов или смене сцены.

### Трансформация

**Трансформация** (она же *transform*) — процесс изменения вида дисплееблов, используя код.

### Трассировка

**Трассировка** (она же *traceback*, *трейсбэк* или *"трейс"*) — отчёт о последних выполненных командах перед возвратом игрой необработанного исключения (ошибки).

### NVL- и ADV-режимы

Режимы вывода текста в игре. В **NVL-режиме** (от англ. *Novel* — новелла) текст выводится сплошным полотном, без дополнительных функций. В **ADV-режиме** (от англ. *Adventure*) текст выводится мелкими частями и может иметь дополнительные интерактивные элементы.

## Термины, используемые в программировании

Далее будут описаны термины, которые зачастую используются и вне пространства написания модов для Бесконечного Лета.

### Костыль

**Костыль** — быстрое и, довольно часто, кривое решение проблемы, которая обычно решается довольно долго.

### Префикс и постфикс

**Префикс** (он же *приставка*) — значимая часть слова, стоящая перед корнем слова.

**Постфикс** (он же *суффикс*) — значимая часть слова, стоящая после корня слова.

### Переменная

**Переменная** — область в памяти, содержащая какие-либо данные. В Ren'Py значения переменных не сохраняются после выхода из игры, если это не постоянные (persistent) переменные.

### Постоянная переменная

**Постоянная переменная** (она же *персистент-переменная*, *persistent variable* или просто *persistent*) — переменная, сохраняющая свои данные в отдельный файл для использования после выхода из игры.

### Типы данных

У значения каждой переменной есть свой тип данных:
- **Логический тип** или **булево** (*boolean*, *bool*) — принимает только одно из двух значений: истина или ложь (True или False, в Ren’Py и Python обязательно с большой буквы!);
- **Целочисленный тип** (*integer*, *int*) — принимает значение в виде любого целого числа (без дробной части);
- **Число с плавающей точкой** (*float*) — принимает значение любого числа, в т.ч. дроби;
- **Строковый тип** (*string*, *str*) — принимает значение в виде некоторой последовательности символов;
- **Список** (*list*) — принимает несколько значений и сохраняет их в виде одномерного массива (по одному значению);
- **Кортеж** (*tuple*) — принимает несколько значений, сохраняет их в виде одномерного массива (по одному значению) и препятствует их изменению;
- **Словарь** (*dict*) — принимает несколько значений и сохраняет их в виде массива с ключом к каждому элементу (по два связанных значения);
- **Байты** (*bytes*) — принимает значение в виде последовательности символов и позволяет конвертировать его в чистые байты и наоборот.

### Функция

**Функция** (*function*, *func*) — объект, который может принимать аргументы и возвращать значение.
